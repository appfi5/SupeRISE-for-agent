### 增加 USDC 与币种限额能力

状态：Accepted

目的：

- 扩展 Ethereum 侧可用资产范围
- 增加 server 侧风险控制能力，同时不增加 Agent 使用复杂度

变更：

- 新增 Ethereum 上的 USDC 支持
- 明确已支持币种都必须同时支持余额查询和转账
- 当前支持币种更新为 CKB、ETH、USDT、USDC
- 新增按币种独立配置的日、周、月限额
- 限额由 Owner 在 Owner Mode 中设置，由 server 执行
- Agent 平时不感知限额，仅在触发时收到 `limit` 信息
- Owner 操作不受该套限额约束
- 限额按 server 本地时区重置

## 2026-03-13

### 增加地址簿能力

状态：Accepted

目的：

- 让 Agent 能通过名称复用常用收款目标
- 降低重复输入原始地址带来的使用负担和误操作风险
- 让 Owner 在人工接管时也能沿用同一套联系人能力
- 让 Agent 在面对精确地址输入时识别地址簿中的已知联系人

变更：

- 新增地址簿能力，用于以名称管理常用收款目标
- 同一个联系人名称可分别记录 Nervos 地址和 Ethereum 地址
- 同一个联系人名称在同一条链下只允许记录一个地址
- 同一条链下的同一个地址可以被多个联系人名称记录
- 系统不对同一个地址挂在多个联系人下做限制
- Agent 可新增、查看、修改和删除联系人
- Agent 可查看联系人列表、单个联系人完整信息和全部联系人完整信息
- Agent 可通过精确地址查询地址簿中的匹配联系人
- 匹配到多少联系人，就返回多少联系人
- 新增或修改联系人时，系统会校验所提供地址是否合法
- 转账时支持直接使用联系人名称作为收款目标
- 系统会按当前转账链自动解析联系人名称到正确地址
- 按精确地址查询只表达地址簿中的匹配联系人，不表达链上真实归属
- 当联系人名称不存在或当前链下未记录地址时，系统会明确返回失败
- Owner Mode 提供与 Agent 对应的地址簿操作支持

## 2026-03-17

### 增加产品侧多语言支持

状态：Accepted

目的：

- 降低不同语言用户的使用门槛
- 让 Owner 在人类介入时能够以熟悉的语言使用产品

变更：

- 在产品侧新增英文和中文两种语言支持
- 默认语言设为英文
- 当前多语言支持覆盖 Owner Mode UI 中的固定界面文案和产品提示文案
- Owner 可以使用英文或中文界面完成主要操作
- 当前不支持英文和中文之外的其他语言
- 用户自行输入的内容不属于本期多语言翻译范围
