# Multi-language Support

状态：Accepted
更新时间：2026-03-17
已回写：[`prd.md`](../prd.md)

## 为什么有这份文档

这份文档用于定义多语言支持的需求边界，并补充正式 PRD 未展开的 feature 说明。

当前产品已具备 Agent 使用入口和 Owner Mode UI，但在人类介入场景下，如果产品只能以单一语言提供固定界面文案和提示信息，会直接增加使用门槛。

因此，本次多语言支持的目标不是做完整的全球化体系，而是在产品侧先提供明确、可用的中英文支持，降低 Owner 的理解和操作成本。

## 核心诉求

本次多语言支持的核心诉求如下：

1. 在产品侧支持英文和中文两种语言
2. 默认语言为英文
3. Owner 能以英文或中文完成主要操作
4. 固定界面文案和产品提示文案应随所选语言展示

## 产品定位

本次多语言支持是产品可理解性能力，而不是：

- 新的业务能力
- 链上数据翻译能力
- 用户输入内容翻译能力
- 英文和中文之外的完整多语言体系

它的职责是：

- 降低不同语言用户的使用门槛
- 支持 Owner 在人类介入时以熟悉语言使用产品
- 让固定界面文案和风险提示以一致语言呈现

## 需求边界

### 1. 当前支持语言

本期只支持以下两种语言：

- 英文
- 中文

本期不扩展到其他语言。

### 2. 默认语言

本期默认语言应为英文。

也就是说：

- 产品在默认状态下应以英文展示固定界面文案和产品提示文案

### 3. 适用范围

本期多语言支持应覆盖：

- Owner Mode UI 中的固定界面文案
- 产品中的固定提示文案
- 风险提示文案

### 4. 不属于本期范围的内容

本期不要求翻译以下内容：

- 用户自行输入的联系人名称
- 用户自行输入的备注
- 地址、交易标识等原始数据
- 资产符号等标准标识

## 核心场景

### 1. Owner 使用默认英文界面

在默认状态下，Owner 进入产品时，应看到英文界面和英文提示文案。

### 2. Owner 使用中文界面

Owner 在需要时，应能够使用中文界面完成主要操作，而不必依赖英文理解产品。

## 已确认的正式需求方向

本次已确认的正式需求，至少应落成以下几类正式需求：

1. 产品侧英文和中文支持
2. 默认语言为英文
3. 固定界面文案和产品提示文案的双语覆盖
4. 用户输入内容不纳入翻译范围
